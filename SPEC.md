# Eggchi 最新仕様書

このドキュメントは、現行の [index.html](C:\Users\moyas\source\eggchi\index.html) と [sw.js](C:\Users\moyas\source\eggchi\sw.js) に基づく最新版の仕様書です。

## 1. ペットの基本ステータスと減少ロジック

### 管理ステータス

- 管理ステータスは `おなか`、`なかよし`、`たいりょく` の3種類です。
- いずれも内部的には `0〜100` の数値で管理します。
- 内部には追加で `sickness` という病状メーターがあり、見た目の病気フラグ `needsTreatment` と組み合わせて使います。

### 通常時の減少量

- 起床中
- `おなか`: `0.085 / 秒` 減少
- `なかよし`: `0.06 / 秒` 減少
- `たいりょく`: `0.045 / 秒` 減少

### 睡眠時の変化

- 睡眠中
- `おなか`: `0.004 / 秒` 減少
- `なかよし`: `0.002 / 秒` 減少
- `たいりょく`: `0.11 / 秒` 回復

### ウンチの発生

- 年齢が `60秒` を超えるとウンチが発生し始めます。
- 起床中はだいたい `15分` 単位で増えます。
- 睡眠中はかなりゆっくり増えます。
- ウンチの保持数は最大 `3個` です。

### 病気の進行

- ウンチが `2個以上` ある状態を `3時間` 放置すると病気になります。
- 病気になると `needsTreatment=true` になります。
- 病気中は `ごはん` と `あそぶ` ができません。
- 病気中は `そうじ` または `ちりょう` で回復できます。
- `sickness` は、放置や危険状態で増え、快適状態では少しずつ下がります。

### 不機嫌の条件

- 次のいずれかで不機嫌になります。
- ウンチが `2個以上`
- 食べ残しを `1時間以上` 放置
- `5分以内に3回` 以上たたき起こす

### 不機嫌中のペナルティ

- `なかよし` の増加量が通常の約 `45%` になります。
- `なかよし` の上限が `80` になります。
- 時間経過中も、起床中は追加で `なかよし` が少し下がりやすくなります。

### 危険状態と死亡

- 次のいずれかで危険状態になります。
- `sickness >= 100`
- `おなか <= 0`
- `なかよし <= 0`
- 起床中に `たいりょく <= 0`
- 危険状態が `6時間` 続くと死亡します。
- 死亡後は中央ボタンで新しいタマゴにリセットできます。

## 2. 睡眠・起床のルール

### 睡眠開始

- `ねかせる` を押すと睡眠状態に入ります。
- 睡眠開始時に `たいりょく +18` されます。
- 睡眠時間は `8時間` です。

### 睡眠中の制限

- 睡眠中は `ごはん` と `あそぶ` を実行できません。
- 実行しようとするとフィードバックを表示してキャンセルします。
- 睡眠中はアクション表示が `おこす` に切り替わります。

### 起床

- `おこす` を押すと手動で起こせます。
- `8時間` 経過しても自動的に起きます。

### 無理やり起こした時のペナルティ

- 手動起床時は起床履歴を記録します。
- `5分以内に3回` 起こすと `30分` の不機嫌になります。

## 3. ミニゲーム5種類

共通仕様

- `あそぶ` を押すと、5種類のミニゲームからランダムで1つが始まります。
- 毎回 `3, 2, 1, START!` のカウントダウンがあります。
- ミニゲーム終了時はスコアに応じて `なかよし` が上がります。
- ミニゲーム後は `たいりょく` と `おなか` が少し減ります。

### 3.1 ほしキャッチ

- 制限時間: `6秒`
- ルール: 飛び回る星をタップする
- 1回タップ成功で `+1点`
- 星は約 `650ms` ごとにランダム位置へ移動します
- 成功判定
- `12点以上`: 大成功
- `7点以上`: 成功

### 3.2 リズムストップ

- 制限時間ではなく `3回チャレンジ制`
- 横に動くバーをタイミングよく止めるゲームです
- バーは細い中央ターゲットに向かって往復します
- 押した瞬間の実際の位置でバーが止まります
- 中心との差で評価とスコアが決まります

評価と加点

- `すごい！`: 中心差 `1.2以下`、`+12点`
- `いいね！`: 中心差 `3以下`、`+8点`
- `おしい！`: 中心差 `6以下`、`+4点`
- `ざんねん…`: それ以上、`+1点`

追加演出

- 評価メッセージを約 `1秒` 表示します
- バーは止まった位置に残るため、ズレが視覚的に分かります

成功判定

- 3回合計 `24点以上`: 大成功
- 3回合計 `14点以上`: 成功

### 3.3 ゴミよけダッシュ

- 制限時間: `6秒`
- 3レーン制
- 操作は `左` と `右` の2ボタンのみ
- ボタンを押すと1段階ずつ左右に移動します
- 障害物は2個同時に落下し、出現レーンはランダムです

得点

- 障害物を避けて画面下まで落とすと `+2点`
- 自分のいるレーンでぶつかると `-2点`

成功判定

- `10点以上`: 大成功
- `5点以上`: 成功

### 3.4 パワーチャージ

- 制限時間: `6秒`
- ルール: 中央の `TAP` ボタンを連打してパワーをためる
- 1タップごとに `+1点`

成功判定

- `26点以上`: 大成功
- `14点以上`: 成功

### 3.5 ひかりおぼえ

- 制限時間ではなく `3回チャレンジ制`
- 各ラウンドで `ド / レ / ミ` の3ボタンが順番に光ります
- プレイヤーは同じ順で押します
- 再生時にも入力時にも、ボタンごとに異なる音程が鳴ります

ラウンド構成

- 1回目: 長さ `3`
- 2回目: 長さ `4`
- 3回目: 長さ `5`

得点

- ラウンド成功: `長さ × 2点`
- ラウンド失敗: `-1点`

成功判定

- `18点以上`: 大成功
- `10点以上`: 成功

## 6. 追加された特殊ギミック

### 食べ残し

- `おなか >= 95` のときにごはんをあげても食べません
- 一時メッセージで `おなかがいっぱいでたべられないみたい` を表示します
- 食べられなかったごはんは床の食べ残しになります
- 食べ残しは最大 `3個`
- `1時間` 放置すると不機嫌の原因になります
- `そうじ` でまとめて片付けられます

### 病気中専用の治療

- `ちりょう` ボタンは病気のときだけ表示されます
- 実行すると `needsTreatment` を解除し、`sickness` を大きく下げます

### サウンド演出

- 外部音声ファイルは未使用です
- Web Audio API で生成しています
- 実装済みの音

- ファンファーレ

### 成功リアクション

## 7. PWA / オフライン対応

### 現在のPWA構成

- `manifest.webmanifest` を設定済み
- `display: standalone`
- `orientation: portrait`
- アイコン
- `icons/icon-192.svg`
- `icons/icon-512.svg`

### Service Worker

- `sw.js` で Service Worker を登録済みです
- `index.html`、`manifest.webmanifest`、アイコン2種をアプリシェルとして事前キャッシュします
- ナビゲーション要求は基本的にネットワーク優先です
- 通信失敗時はキャッシュ済みの `index.html` を返します
- 同一オリジンの GET リクエストはランタイムキャッシュにも保存します
- 古いキャッシュは `activate` 時に削除します

### オフライン動作

- 一度オンラインで開いてキャッシュが完了した端末では、PWAとしてオフライン起動しやすい構成です
- 音は Web Audio API 生成なので、音声ファイルの個別キャッシュは不要です

### 運用上の前提

- 恒久運用は GitHub Pages などの常時 HTTPS ホスティングを前提にするのが最も安定です
- Service Worker のみでも端末側キャッシュで起動しやすくなりますが、公開URLが常に存在するホスティング構成の方が確実です
