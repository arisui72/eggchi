# Lumini 最新仕様書

このドキュメントは、現行の [index.html](C:\Users\moyas\source\eggchi\index.html) と [sw.js](C:\Users\moyas\source\eggchi\sw.js) に基づく最新版の仕様書です。

## 1. アプリ概要

- アプリ名は `Lumini` です。
- 1画面完結のバーチャルペット育成 Web アプリです。
- データ保存は `LocalStorage` を利用します。
- PWA 対応済みで、ホーム画面追加時の standalone 表示を前提にレイアウトを最適化しています。
- セーブキーは `lumini-pocket-save-v1` です。

## 2. 基本ステータスと減少ロジック

### 管理ステータス

- 管理ステータスは `おなか`、`なかよし`、`たいりょく` の 3 種類です。
- すべて `0〜100` の数値で管理します。
- 内部には補助パラメータとして `sickness` があり、見た目の病気フラグ `needsTreatment` と組み合わせて使います。

### 通常時の減少量

- 起床中
- `おなか`: `0.085 / 秒` 減少
- `なかよし`: `0.06 / 秒` 減少
- `たいりょく`: `0.045 / 秒` 減少

### 睡眠時の変化

- 睡眠中
- `おなか`: `0.004 / 秒` 減少
- `なかよし`: `0.002 / 秒` 減少
- `たいりょく`: `0.11 / 秒` 回復

### ウンチ・食べ残し・病気

- 年齢が `60秒` を超えるとウンチが発生し始めます。
- 起床中はおおむね `15分` 単位、睡眠中はかなりゆっくり増えます。
- ウンチの保持数は最大 `3個` です。
- `おなか >= 95` のときにごはんをあげると食べず、床の食べ残しとして最大 `3個` までたまります。
- 食べ残しを `1時間` 放置すると不機嫌の原因になります。
- ウンチが `2個以上` ある状態を `3時間` 放置すると病気になります。
- 病気中は `ごはん` と `あそぶ` ができません。
- `そうじ` または `ちりょう` で回復できます。

### 不機嫌の条件

- 次のいずれかで不機嫌になります。
- ウンチが `2個以上`
- 食べ残しを `1時間以上` 放置
- `5分以内に3回` 以上たたき起こす

### 不機嫌中のペナルティ

- `なかよし` の上昇量が通常の約 `45%` になります。
- `なかよし` の上限が `80` になります。
- 表情も不満そうな専用顔になります。

### 危険状態と死亡

- 次のいずれかで危険状態になります。
- `sickness >= 100`
- `おなか <= 0`
- `なかよし <= 0`
- 起床中に `たいりょく <= 0`
- 危険状態が `30分` 続くと死亡します。
- 死亡時はお墓イラストと専用メッセージを表示します。
- `つぎの世代をそだてる` ボタンで、図鑑を引き継いだまま `1だいめ` から再スタートできます。

## 3. 睡眠・起床

- `ねかせる` を押すと睡眠状態に入ります。
- 睡眠開始時に `たいりょく +18` されます。
- 睡眠時間は `8時間` です。
- 睡眠中は `ごはん` と `あそぶ` が実行不可です。
- 睡眠中はアクション表示が `おこす` に切り替わります。
- `おこす` を押すか、`8時間` 経過すると起床します。
- `5分以内に3回` 起こすと `30分` の不機嫌になります。

## 4. お世話アクション

- `ごはん`
- `あそぶ`
- `そうじ`
- `ちりょう`
- `ねかせる / おこす`

### ごはん

- 通常時は `おなか +24`、`なかよし` も少し上がります。
- そのかわりウンチが増えやすくなります。
- `おなか >= 95` のときは食べず、食べ残しが増えます。

### あそぶ

- 5種類のミニゲームからランダムで1つ始まります。
- ミニゲーム後はスコアに応じて `なかよし` が上がります。
- `たいりょく` と `おなか` は少し減ります。
- `たいりょく <= 0` のときは遊べません。

### そうじ

- ウンチ、食べ残し、病気フラグをまとめて解消します。
- `sickness` も少し下げます。

### ちりょう

- 病気のときだけ表示されます。
- `needsTreatment` を解除し、`sickness` を大きめに下げます。

## 5. ミニゲーム

### 共通仕様

- `あそぶ` 実行時に 5 種からランダムで 1つ選ばれます。
- 毎回 `3, 2, 1, START!` のカウントダウンがあります。
- READY 中の説明文はゲームごとに切り替わります。

### 5.1 ほしキャッチ

- 制限時間: `6秒`
- 星をタップすると `+1点`
- 星は約 `650ms` ごとにランダム位置へ移動
- `12点以上`: 大成功
- `7点以上`: 成功

### 5.2 リズムストップ

- `3回チャレンジ制`
- 横移動するバーを細い中央ターゲットのど真ん中で止めるゲームです。
- 押した瞬間の実位置でバーが止まります。

評価と加点

- `すごい！`: 中心差 `1.2以下`、`+12点`
- `いいね！`: 中心差 `3以下`、`+8点`
- `おしい！`: 中心差 `6以下`、`+4点`
- `ざんねん…`: それ以上、`+1点`

成功判定

- 3回合計 `24点以上`: 大成功
- 3回合計 `14点以上`: 成功

### 5.3 ゴミよけダッシュ

- 制限時間: `8秒`
- 3レーン制
- 操作は `左` と `右` の 2 ボタン
- ボタンを押すと 1段階ずつ移動
- 障害物は 2 個同時に落下し、出現レーンはランダム
- 回避成功で `+2点`
- 接触で `-2点`
- 接触時は軽いダメージ音が鳴ります
- `16点以上`: 大成功
- `9点以上`: 成功

### 5.4 パワーチャージ

- 制限時間: `6秒`
- `TAP` ボタン連打で `+1点`
- `26点以上`: 大成功
- `14点以上`: 成功

### 5.5 ひかりおぼえ

- `3回チャレンジ制`
- `ド / レ / ミ` の順番を記憶して再現します
- 再生時も入力時もボタンごとに別音程が鳴ります

ラウンド構成

- 1回目: 長さ `3`
- 2回目: 長さ `4`
- 3回目: 長さ `5`

得点

- 成功: `長さ × 2点`
- 失敗: `-1点`

成功判定

- `18点以上`: 大成功
- `10点以上`: 成功

## 6. 進化

### 基本ルール

- 進化は自動確定ではありません。
- 条件達成で「光って待機」状態になります。
- `しんか！` ボタンを押すと進化が確定します。

### 進化年齢

- `teen`: `60分`
- `adult`: `300分`
- `elder`: `600分`
- `mythic`: `900分`

### 飛び級防止

- 各段階に `stageStartedAt` を持ちます。
- 一段進化するたびに次の進化時計をリセットします。
- 長時間アプリを閉じていても、一度に一段階ずつしか進化しません。

### 分岐系統

- `spark`: 遊び寄り
- `cozy`: お世話・睡眠寄り
- `munch`: ごはん寄り

### 分岐に使う主な要素

- `spark`: `play`、`gameWins`、`なかよし`
- `cozy`: `clean`、`sleep`、`たいりょく`、病気の少なさ
- `munch`: `feed`、`おなか`、部屋の清潔さ

### 進化先一覧

- 初期: `たまごベビー`
- spark 系: `ビビッドスパーク` → `ネオンミーティア` → `プリズムライダー` → `サンバーストゼニス`
- cozy 系: `おふとんミスト` → `ムーンナース` → `ドリーミィラグーン` → `セレスティアルシエスタ`
- munch 系: `もぐぱふプリン` → `ボンボンベリー` → `キャラメルグリッター` → `ロイヤルパフェリーヌ`

## 7. 結婚・世代交代

- 最終進化 `mythic` 到達時に `finalFormAt` を記録します。
- 最終進化後 `10時間` で結婚待機条件を満たします。
- 条件達成後は自動実行ではなく、`けっこん！` ボタンで確定します。
- 結婚演出後、数秒で次世代のタマゴへ移行します。
- 世代交代時は `generation` が `+1` されます。
- 図鑑データは世代交代後も維持されます。

## 8. 図鑑

- `ずかん` ボタンからオーバーレイ表示します。
- 未解放キャラは `？？？？` 表示です。
- 進化確定時に該当キャラが図鑑解放されます。
- スクロール対応済みです。
- `× 閉じる` ボタンで閉じられます。
- 図鑑の解放状況は `LocalStorage` に保持されます。

## 10. レイアウトと表示モード

- `display-mode: standalone` と `navigator.standalone` で PWA 起動を判定します。
- `body.is-standalone` と `body.is-browser` を切り替えます。
- レイアウトの基準は PWA です。
- PWA 時はペットエリアを広め、下部ボタン群を押しやすく配置します。
- ブラウザ表示時はアドレスバーやツールバーぶんを見込んだ少し詰めたレイアウトになります。
- ダブルタップ拡大を避けるため、`viewport` と `touch-action` を調整しています。

## 11. セーブと復旧

- 重要なタイミングで `LocalStorage` に自動保存します。
- 定期セーブも有効です。
- 読み込み時は `sanitizeState()` で異常値を補正します。
- 壊れたセーブデータでも安全な初期値へフォールバックします。
- 実行時エラー時は簡易トーストを表示し、復旧可能な範囲で継続します。

## 12. PWA / オフライン対応

### manifest

- `manifest.webmanifest` を設定済み
- `display: standalone`
- `orientation: portrait`
- アプリ名は `Lumini`

### Service Worker

- `sw.js` を登録済み
- `index.html`
- `manifest.webmanifest`
- `icons/icon-192.svg`
- `icons/icon-512.svg`

をアプリシェルとして事前キャッシュします。

- ナビゲーションはネットワーク優先
- 失敗時はキャッシュ済み `index.html` を返します
- 同一オリジンの GET はランタイムキャッシュ対象です
- 古いキャッシュは `activate` 時に削除します

### オフライン動作

- 一度オンラインで開いてキャッシュが完了した端末では、PWA としてオフライン起動しやすい構成です。
- 音は Web Audio API 生成のため、外部音声ファイルのキャッシュは不要です。
